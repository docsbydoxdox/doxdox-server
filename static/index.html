doxdox
A simple solution for auto generated documentation.

doxdox is documentation generator that takes JSDoc-style comment blocks from JavaScript files and formats them in to easy to read HTML, Markdown and Dash documentation files.

Get Started Now

Installing doxdox

doxdox is a command line utility written in Node.js and installed via npm.

The following command will install doxdox globally on your computer, allowing you to access it from any project.

        $ npm install doxdox -g

Alternatively you can install it on a per-project basic.

        $ npm install doxdox --save

Getting Started

Basic Usage

Running doxdox without any options will crawl the current directory looking for JavaScript files1 and generate documentation from the files it finds containing valid JSDoc comment blocks.

doxdox will also search the current directory for a package.json file in order to populate both the title and description of the generated file.

Output Contents to stdout

        $ doxdox

Output Contents to Destination File

This can be achieved by using the cat shorthand operator > or the output flag.

        $ doxdox > docs/index.html
        
        $ doxdox --output docs/index.html

Specify Source

Instead of allowing doxdox to crawl the current directory you can specify a single JavaScript file or different directory.

        $ doxdox ~/git/github/Facade.js/facade.js
        
        $ doxdox ~/git/github/Facade.js/

Advanced Usage

Set Title

        $ doxdox --title "Facade.js"

Set Description

        $ doxdox --description "Drawing shapes, images and text in HTML5 canvas made easy."

Choose a Layout

        $ doxdox --layout Bootstrap
        
        $ doxdox --layout Markdown
        
        $ doxdox --layout Dash

Use a Custom Layout

Layouts are built using Handlebars. The included layouts, Bootstrap and Markdown, can be viewed on GitHub.

        $ doxdox --layout docs/template.hbs

Point to a Specific package.json File

        $ doxdox --package ~/git/github/Facade.js/package.json

doxdox Online

doxdox is also available as a PaaS for single file libraries hosted on GitHub.

To use this service add the following snippet to your README file. Replace username and repo with the path of the project on GitHub.

        [![Latest Documentation](http://doxdox.org/images/badge.svg)](http://doxdox.org/username/repo)

You must also have a valid package.json file in your project that contains title, description and main key/value pairs.

        {
            "name": "facade.js",
            "description": "Drawing shapes, images and text in HTML5 canvas made easy.",
            "main": "facade.js"
        }

Notice: This service caches documentation for up to 10 minutes.

Examples

  • Facade.js - Drawing shapes, images and text in HTML5 canvas made easy.
  • plastick.js - Plastick.js is a lightweight, time-accurate game loop framework for JavaScript, utilizing Facade.js.
  • CanvasToVideo - An experiment in converting Canvas animations to video.

  1. ↩ doxdox will ignore the following directories: .git, node_modules, bower_components, and test. It will also ignore files matching .min, gruntfile, and gulpfile.
